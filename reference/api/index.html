    API Reference

    Core Functions

    connect_to_socketcan(config: SocketcanConfig) -> SocketcanFd

    Creates a socketcan socket according to the provided configuration.

    Parameters: - config (SocketcanConfig): Configuration for the CAN connection

    Returns: - SocketcanFd: A socket configured for CAN communication

    Example: 

    from sockcan import SocketcanConfig, connect_to_socketcan

config = SocketcanConfig(channel="can0")
sock = connect_to_socketcan(config)

    build_send_func(fd: SocketcanFd, *, expects_msg_cls: bool = False) -> SendFn | MessageSendFn

    Builds a send function for the given socket.

    Parameters: - fd (SocketcanFd): The socket to send messages on - expects_msg_cls (bool): If True, the function accepts CanMessageProtocol objects; otherwise accepts arbitration_id, data, and is_extended separately

    Returns: - SendFn: A function accepting (arbitration_id, data, is_extended) if expects_msg_cls=False - MessageSendFn: A function accepting a CanMessageProtocol object if expects_msg_cls=True

    Example: 

    # Simple send
send_func = build_send_func(sock, expects_msg_cls=False)
send_func(0x123, b'\x01\x02\x03\x04', False)

# Message class send
send_func = build_send_func(sock, expects_msg_cls=True)
message = CanMessage(arbitration_id=0x456, data=b'\x05\x06\x07\x08', is_extended_id=False, timestamp=0.0)
send_func(message)

    build_recv_func(fd: SocketcanFd, *, use_native_timestamps: bool = True, is_stream: bool = False) -> RecvFn

    Builds a receive function for the given socket.

    Parameters: - fd (SocketcanFd): The socket to receive messages from - use_native_timestamps (bool): Use hardware timestamps when available (default: True) - is_stream (bool): Use stream mode for daemon-based connections (default: False)

    Returns: - RecvFn: A callable that returns a CanMessage when called

    Example: 

    recv_func = build_recv_func(sock)
message = recv_func()
print(f"Received: ID={message.arbitration_id:#x}, Data={message.data.hex()}")

    Data Classes

    SocketcanConfig

    Configuration options for a SocketCAN connection.

    Attribute Type Default Description
    channel str "can0" CAN interface channel name
    loopback LoopbackMode LoopbackMode.FOR_OTHER_SOCKS Loopback mode for sent messages

    CanMessage

    Container for CAN message data.

    Attribute Type Description
    arbitration_id int CAN message arbitration ID
    data bytes CAN message payload data
    is_extended_id bool Whether the arbitration ID is extended (29-bit)
    timestamp float Message timestamp (seconds since epoch)

    LoopbackMode

    Controls whether transmitted messages are received by the sender or other sockets.

    Value Description
    OFF Sent messages are not received by any socket
    FOR_OTHER_SOCKS (Default) Sent messages are received by other sockets on the same CAN device, but not the sender
    ON Sent messages are received by all sockets including the sender

    Daemon Classes

    SocketcanServer

    Listens to a CAN bus and dispatches messages to subscribed consumers.

    Constructor: 

    SocketcanServer(
    bus: BusABC | None = None,
    *,
    use_native_timestamps: bool = False,
    use_stream: bool = False,
)

    Key Methods: - subscribe(filters: set[int] | None = None) -> SocketcanFd: Subscribe to the bus, returns a socketcan-like socket - start(direction: ServerDirection = ServerDirection.BIDIRECTIONAL): Start TX/RX threads - stop(): Stop the server

    Example: 

    from sockcan.daemon import SocketcanServer, BusParameters

with SocketcanServer.factory(BusParameters(channel="can0", interface="pcan")) as server:
    sock = server.subscribe()
    # Use sock with build_send_func and build_recv_func

    SocketcanDaemon

    HTTP-based daemon that provides SocketCAN service over the network.

    Constructor: 

    SocketcanDaemon(host: str = "localhost", port: int = 8000)

    Key Methods: - register_bus(channel, interface, bitrate=500000, *, use_native_timestamps=False): Register a real CAN bus - register_virtual_bus(channel="vcan0"): Register a virtual CAN bus - start(): Start the daemon - stop(): Stop the daemon

    BusParameters

    Parameters for configuring a CAN bus connection.

    Attribute Type Default Description
    channel str "PCAN_USBBUS1" CAN interface channel
    interface str "pcan" python-can interface name
    bitrate int 500000 CAN bus bitrate in bits/second

    ServerDirection

    Controls which directions the socketcan server operates in.

    Value Description
    RX_ONLY Only receive messages from the bus
    TX_ONLY Only transmit messages to the bus
    BIDIRECTIONAL (Default) Both receive and transmit

    Interoperability Functions (python-can Integration)

    hijack_python_can(system: str | None = None) -> None

    Permanently overrides python-can's socketcan implementation with Sockcan's faster version.

    !!! warning This function modifies global shared state in python-can.

    Example: 

    import can
from sockcan.interop import hijack_python_can

hijack_python_can()

# Now uses Sockcan's optimized implementation
bus = can.Bus(interface="socketcan", channel="can0")

    override_python_can(bus_cls: type[FastSocketcanBus | UserspaceSocketcanBus] = FastSocketcanBus) -> contextmanager

    Temporarily overrides python-can's implementation within a context scope.

    Example: 

    from sockcan.interop import override_python_can

with override_python_can():
    # python-can uses Sockcan here
    bus = can.Bus(interface="socketcan", channel="can0")

    activate_userspace_socketcan(parameters, config=None, *, system=None) -> None

    Enables userspace SocketCAN on non-Linux platforms (primarily Windows).

    !!! warning This function modifies global shared state in python-can.

    Example: 

    from sockcan.daemon import BusParameters
from sockcan.interop import activate_userspace_socketcan, SocketcanDaemonConfig

config = SocketcanDaemonConfig(
    mode="daemon",
    allow_run_daemon_locally=True,
)
activate_userspace_socketcan(
    BusParameters(channel="PCAN_USBBUS1", interface="pcan"),
    config=config,
)

    SocketcanDaemonConfig

    Configuration for the userspace SocketCAN daemon mode.

    Attribute Type Default Description
    mode ServerMode "local" Operating mode: "local" (thread) or "daemon" (HTTP server)
    allow_run_daemon_locally bool True Allow starting the daemon in the current process
    host str "localhost" Daemon HTTP server host
    port int 8000 Daemon HTTP server port
    linux_too bool False Allow userspace mode on Linux
    use_native_timestamps bool False Use hardware timestamps
